Лабораторна робота №5



&nbsp;Опис проекту



Кешуючий проксі-сервер для API http.cat. Сервер автоматично зберігає отримані зображення у локальному кеші, що прискорює повторні запити та зменшує навантаження на зовнішній API.



&nbsp;Основні функції



\- \*\*GET\*\* - отримання зображення (з кешу або http.cat)

\- \*\*PUT\*\* - ручне завантаження файлу в кеш

\- \*\*DELETE\*\* - видалення файлу з кешу



&nbsp;Структура проекту

```

backend-course-2025-5/

├── main.js           # Головний файл сервера

├── package.json      # Конфігурація та залежності

├── .gitignore       # Ігноровані файли

├── test.txt         # Команди для тестування

└── cache/           # Директорія кешу

```



&nbsp;Використані технології



\- \*\*Node.js\*\* (node:http, node:fs)

\- \*\*Commander.js\*\* - парсинг аргументів

\- \*\*Superagent\*\* - HTTP-запити



&nbsp;Запуск

```bash

node main.js -h localhost -p 3000 -c cache

```



&nbsp;Приклади використання

```bash

&nbsp;Отримати зображення

curl http://localhost:3000/200 -o test.jpg



&nbsp;Видалити з кешу

curl -X DELETE http://localhost:3000/200



&nbsp;Завантажити своє зображення

curl -X PUT http://localhost:3000/999 --data-binary "@image.jpg"

```
